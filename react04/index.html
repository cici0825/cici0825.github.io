React 的渲染機制與 VirtualDOM

16 April 2020 — Written by WuFangCi
#React#useEffect#useLayoutEffect#week21#VirtualDOM#useCallback#memo

參考資料 實作一個簡易 Virtual DOM 吧！

透過實作我們都是更改狀態都是直接對 state 做改變，每當 state 一有動靜 react 就會幫我們 re-render，那 react 的底層是透過什麼方式讓 render 機制變得有效率呢？更精確的問，針對 react 的機制，react 如何快速找到要改變的地方？

WHY Virtual DOM

  • 在目前的網頁中DOM的數目遠遠超過我們練習的作業龐大，當DOM的數目越高，更動DOM的成本也越高，並且會降低網頁上的效能，也可能會增加很多 bug。
  • 透過 Virtual DOM ，不直接操作 DOM，而是將頁面上的 DOM 經 parse/traversal 為 JavaScript 物件暫存在某個地方，對這些物件操作，然後再更新到 DOM 上，想必會比直接操作 DOM 來得快速許多。因此，使用 Virtual DOM 的最大好處就是提升效能 。

備註：Virtual DOM 保證的並不是在所有狀況下都比操作原本的 DOM 來得快速，而是在普遍狀況下，不需經手動優化，仍能給予不錯的效能。

==原本的 APP.js 程式碼==

  • 繼 todo-list 示範
function App() {
  const {
    id,
    todos,
    setTodos,
    value,
    setValue,
    handleChange,
    handelButtonClick,
    handleToggleIsDone,
    handleDeleteTodo,
  } = useTodos();

  return (
    <div className="APP">
      <input
        type="text"
        placeholder="add todo..."
        value={value}
        onChange={handleChange}
      />
      <button onClick={handelButtonClick}> Add Todo </button>
      {todos.map((todo) => (
        <BlackTodoItem
          key={todo.id}

          handleToggleIsDone={handleToggleIsDone}

      ))}
    </div>
  );
}

function App() {
  const {
    id,
    todos,
    setTodos,
    value,
    setValue,
    handleChange,
    handelButtonClick,
    handleToggleIsDone,
    handleDeleteTodo,
  } = useTodos();

  function Button({ onClick, children }) {
    console.log("render Button");
    return <button onClick={onClick}>{children}</button>;
  }
  return (
    <div className="APP">
      <input
        type="text"
        placeholder="add todo..."
        value={value}
        onChange={handleChange}
      />
      <Button onClick={handelButtonClick}> Add Todo </Button>
      {todos.map((todo) => (
        <BlackTodoItem
          key={todo.id}
          todo={todo}
          handleDeleteTodo={handleDeleteTodo}
          handleToggleIsDone={handleToggleIsDone}
        />
      ))}
    </div>
  );
}

export default App;

拆出來後只要是跟 Button component 掛鉤 的 children ，只要一被更動狀態 Button 就會被 render。但是 Button 跟打字一點關係都沒有，不應該因為打幾個字就重新 render Button。

==使 component 不 re-render 的方法==

使用 #memo (for component)

  • import： import { memo } from "react";
  • 操作：Button 包在 memo 裡 const MemoButton = memo(Button);
  • 原理：react 自動檢查如果你傳遞的 props { onClick ,children} 都沒有改變的話，就不會 re-render

看一下 console

結果還是出現了 re-render，原因是因為我們執行 onClick 事件傳進去的 function 每次執行的時候都會不一樣 ， 因為 call by reference 的關係，所以我們可以用另外一個 hooks 來解決這個問題。

使用 #useCallback

  • import： import { useCallback } from "react";
  • 操作： handleButtonClick 的function 包在 useCallBack 裡。
  • 操作： useCallback(function, []) ，第二個參數為空陣列 當我沒有變動時，function 就不會改變，所以只會在 一開始瀏覽器畫畫面的時候會執行。
  • 可以想成 react 幫我們記住了這個尚未更動的值。
const handelButtonClick = useCallback(() => {
    setTodos([
      {
        id: id.current,
        content: value,
      },
      ...todos,
    ]);

    setValue("");
    id.current++;
  }, []);

再次檢查 console ， 沒有因為打字而 re-render Button 的問題了。

假設錯誤

遇到新增上的 BUG => 當我點擊新增的時候，出現的只剩下空的 todos 。

==檢討原因==： 因為我們在第二個參數 加上 [] 空的陣列，告訴 react 這裡不會有改變。 但是我們點擊事件，的目的是為了新增代辦。 而代辦事項，是來自我們需要取到 input value 的值。 因此點擊事件後， 雖然我們用了 useCallBack 不會重新 render Button 了 但我們得不到任何 value 的值。

為了讓 todolist 能夠順利新增 我們要在 useCallBack 的 第二個參數放進 關注的state。於是我們又回到了 re-render 的狀態 XDD。 就如前面所推，handleButtonClick 是需要 value 的，因為我們選擇 #controller-component {value} ，而不是 #unController value。

  const handelButtonClick = useCallback(() => {
    setTodos([
      {
        id: id.current,
        content: value,
      },
      ...todos,
    ]);
    setValue("");
    id.current++;
  }, [setValue, setTodos, value, todos]);
